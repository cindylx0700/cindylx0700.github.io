Master of Computer & Innovation Project

Visualizing Human Question Answering

The task brief was to develop a system which can display text segments and questions from a hand-labelled question answering dataset. The text segments were colour-coded for a selected question using a provided answer which included basic features of the question, matched by string matching techniques.

Project Introduction

Question answering (QA) is a challenging problem in the fields of Natural Language Processing (NLP) and Information Retrieval (IR), focusing on building systems that automatically answer questions posted by humans. In order to build effective QA systems, it is crucial to understand the complex process of human question answering. This includes the human’s judgment about aspects like question type (e.g. factual, list), question stem (e.g. which, when, why), nouns, verbs, and other information in the question and how these aspects can be incorporated together in determining the ‘answer’.

The proposed system aims to simplify this complex human question answering process by visualising basic features in the question and the corresponding answer, will help to understand the basic features of questions used by humans when finding answers.

This project consisted of myself with one other team member and this was my first web-based project. We followed Agile approach during this project. The first challenge for us was that this project didn't restrit the platform. so we undertook considerable research to ensure the best platform was chosen. It was concluded to achieve a better visualization function, we chose to put it on a web platform. Another challenge for us was the raw data convert and NLP algorithm In order to achieve visualization function better, we decided via web.

I was responsible for the Front-end development and my partner was responsible for Data sorting.When all data was converted into JSON, I use AJAX and jQuery to achieve the web data match.
